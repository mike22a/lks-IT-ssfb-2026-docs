# Homework 04 - Desktop Apps Advanced Features

## Informasi
- **Sesi**: 5
- **Deadline**: Sesi 6
- **Estimasi Waktu**: 2-3 jam
- **Fokus**: Data Visualization, Advanced Validation, Access Control

## Tugas 1: Complete Data Visualization

### Deskripsi
Implementasi chart untuk data visualization menggunakan data dari API.

### Requirements
1. **Add Chart Control ke Form**
   - Add Chart control ke Windows Form
   - Configure chart properties
   - Set chart type (Bar, Line, atau Pie)

2. **Load Data dari API**
   - Load data yang relevan untuk chart (misal: sales per category)
   - Agregasi data jika perlu
   - Bind data ke chart

3. **Customize Chart**
   - Set chart title
   - Configure axis labels
   - Set colors untuk series
   - Add legend jika perlu
   - Format data labels (currency, percentage, dll.)

4. **Test Chart**
   - Verify chart menampilkan data dengan benar
   - Test dengan berbagai data scenarios
   - Verify chart updates saat data berubah

### Deliverables
- [ ] Form dengan Chart control
- [ ] Data loading implementation
- [ ] Chart customization
- [ ] Screenshot chart dengan data

---

## Tugas 2: Advanced Validation

### Deskripsi
Implementasi comprehensive validation untuk semua forms.

### Requirements
1. **Implementasi Field-by-Field Validation**
   - Gunakan ErrorProvider component
   - Validasi saat user input (Validating event)
   - Validasi saat form submit
   - Display error icons dan tooltips

2. **Implementasi Business Rule Validation**
   - Validasi rules spesifik (misal: email unique, phone format)
   - Validasi cross-field (misal: end date >= start date)
   - Validasi dengan API jika perlu (misal: check duplicate)

3. **Implementasi User-Friendly Error Messages**
   - Error messages yang jelas dan spesifik
   - Error messages dalam Bahasa Indonesia
   - Error messages yang actionable

4. **Test Validation**
   - Test dengan valid input
   - Test dengan invalid input
   - Test dengan edge cases
   - Verify error messages yang tepat

### Deliverables
- [ ] Validation implementation dengan ErrorProvider
- [ ] Business rule validation
- [ ] Error messages yang user-friendly
- [ ] Test results

---

## Tugas 3: Access Control (Optional)

### Deskripsi
Implementasi login form dan role-based access control.

### Requirements
1. **Create Login Form**
   - Buat form dengan Username dan Password fields
   - Implementasi login button
   - Validasi credentials (hardcoded atau dari API)
   - Set user role setelah login

2. **Implementasi Role-Based Access Control**
   - Define roles (Admin, User, dll.)
   - Restrict menu items berdasarkan role
   - Restrict form access berdasarkan role
   - Hide/disable controls berdasarkan role

3. **Implementasi Logout**
   - Add logout button
   - Clear user session
   - Return ke login form

4. **Test Access Control**
   - Test login dengan berbagai roles
   - Verify access restrictions
   - Verify menu items yang tepat

### Deliverables
- [ ] Login Form
- [ ] Role-based access control implementation
- [ ] Logout functionality
- [ ] Test results

## Tips

### Data Visualization
- Pilih chart type yang sesuai dengan data
- Gunakan colors yang kontras dan readable
- Label axis dengan jelas
- Consider performance untuk large datasets

### Advanced Validation
- Use ErrorProvider untuk consistent error display
- Validate as-you-go untuk better UX
- Provide clear error messages
- Test edge cases thoroughly

### Access Control
- Keep roles simple untuk awal
- Use enum atau constants untuk role definitions
- Test semua role scenarios
- Consider security implications
