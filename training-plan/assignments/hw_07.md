## Tugas 1: Complete CRUD di Mobile

### Deskripsi
Implementasi CRUD operations lengkap untuk customer di mobile dengan API integration.

### Requirements
1. **Implementasi POST untuk Create Customer**
   - Setup Retrofit untuk POST endpoint
   - Create request model untuk customer
   - Implementasi API call dengan coroutines
   - Handle loading state
   - Handle success dan error states
   - Refresh list setelah create

2. **Implementasi PUT untuk Update Customer**
   - Setup Retrofit untuk PUT endpoint
   - Load customer data ke form untuk edit
   - Implementasi API call dengan coroutines
   - Handle loading state
   - Handle success dan error states
   - Refresh list setelah update

3. **Implementasi DELETE untuk Delete Customer**
   - Setup Retrofit untuk DELETE endpoint
   - Add delete button di item atau detail screen
   - Implementasi confirmation dialog
   - Implementasi API call dengan coroutines
   - Handle success dan error states
   - Refresh list setelah delete

4. **Test Semua CRUD Operations**
   - Test create dengan valid data
   - Test read (load data dari API)
   - Test update dengan berbagai scenarios
   - Test delete dengan konfirmasi
   - Test error scenarios

### Deliverables

- [ ] DELETE implementation

## Tugas 2: Form Validation di Mobile

### Deskripsi
Implementasi comprehensive validation untuk customer form.

### Requirements
1. **Implementasi Field Validation**
   - Validasi CustomerName tidak boleh kosong
   - Validasi Email tidak boleh kosong dan format valid
   - Validasi Phone format (jika diisi)
   - Validasi Address tidak boleh kosong (jika required)

2. **Display Error Messages**
   - Display error message di bawah TextField
   - Gunakan TextField dengan isError parameter
   - Error messages dalam Bahasa Indonesia
   - Error messages yang jelas dan actionable

3. **Disable Submit Button jika Invalid**
   - Validasi form secara real-time
   - Disable Save button jika form invalid
   - Enable Save button hanya jika form valid
   - Atau tampilkan error saat submit

4. **Test Validation Scenarios**
   - Test dengan semua field kosong
   - Test dengan invalid email
   - Test dengan valid data
   - Test dengan partial data

### Deliverables
- [ ] Validation implementation
- [ ] Error message display
- [ ] Submit button logic
- [ ] Test results

## Tugas 3: Navigation

### Deskripsi
Implementasi navigation antar screens dengan data passing.

### Requirements
1. **Setup Navigation Component**
   - Add Navigation Compose dependencies
   - Create NavHost
   - Define routes untuk screens:
     - "customer_list"
     - "customer_form"
     - "customer_detail" (opsional)

2. **Implementasi Navigation antar Screens**
   - Navigate dari list ke form (mode add)
   - Navigate dari list ke form (mode edit)
   - Navigate back setelah save
   - Navigate back dengan cancel

3. **Pass Data antar Screens**
   - Pass customer ID untuk edit mode
   - Pass customer data untuk detail screen (opsional)
   - Handle navigation arguments
   - Parse arguments di destination screen

4. **Implementasi Back Navigation**
   - Handle back button behavior
   - Confirm unsaved changes sebelum back
   - Navigate back ke list screen
   - Handle back stack properly

- [ ] Navigation implementation
- [ ] Data passing implementation

## Tips

### CRUD Operations
- Gunakan viewModelScope untuk coroutines
- Use LaunchedEffect untuk one-time operations
- Handle loading states dengan CircularProgressIndicator
- Handle error states dengan user-friendly messages
- Refresh data setelah operations

### Form Validation
- Gunakan derived state untuk form validity
- Use remember untuk validation state
- Display error messages dengan Text composable
- Test validation dengan berbagai scenarios

### Navigation
- Gunakan NavController untuk navigation
- Define routes dengan string constants
- Use navController.navigate() untuk navigation
- Use navController.popBackStack() untuk back navigation
- Handle navigation arguments dengan NavType
