## Tugas 1: Complete Error Handling

### Deskripsi
Implementasi proper error handling untuk semua endpoints.

### Requirements
1. **Implementasi Try-Catch Blocks**
   - Wrap semua API calls dengan try-catch
   - Catch specific exceptions (SQLException, ArgumentException, dll.)
   - Log errors untuk debugging (jika diizinkan)

2. **Return Appropriate Status Codes**
   - 200 OK untuk successful operations
   - 201 Created untuk successful POST
   - 400 Bad Request untuk invalid input
   - 404 Not Found untuk resource tidak ditemukan
   - 500 Internal Server Error untuk unexpected errors

3. **Create Consistent Error Response Format**
   ```json
   {
     "success": false,
     "error": {
       "code": "ERROR_CODE",
       "message": "Error message description",
       "details": [
         {
           "field": "fieldName",
           "message": "Field-specific error"
         }
       ]
     }
   }
   ```

4. **Test Error Scenarios**
   - Test dengan invalid input
   - Test dengan resource tidak ditemukan
   - Test dengan database errors
   - Test dengan network errors (jika applicable)

### Deliverables
- [ ] Error handling implementation di semua controllers
- [ ] Consistent error response format
- [ ] Test results untuk error scenarios
- [ ] Error handling middleware (jika diperlukan)

## Tugas 2: API Documentation dengan Swagger

### Deskripsi
Setup Swagger dan document semua API endpoints.

### Requirements
1. **Setup Swagger**
   - Install Swashbuckle.AspNetCore NuGet package
   - Configure Swagger di Program.cs
   - Enable Swagger UI
   - Test Swagger UI di browser

2. **Add XML Comments**
   - Enable XML documentation di project
   - Add XML comments untuk semua controllers
   - Add XML comments untuk semua actions
   - Include parameter descriptions
   - Include response descriptions

3. **Document Request/Response Models**
   - Add XML comments untuk model classes
   - Document property descriptions
   - Include data types
   - Include validation rules

4. **Test Swagger UI**
   - Verify semua endpoints muncul di Swagger UI
   - Test endpoints dari Swagger UI
   - Verify request/response examples
   - Verify parameter descriptions

### Deliverables
- [ ] Swagger configuration
- [ ] XML comments untuk semua endpoints
- [ ] XML comments untuk semua models
- [ ] Screenshot Swagger UI dengan documentation
- [ ] Test results dari Swagger UI

## Tugas 3: Advanced Business Logic

### Deskripsi
Implementasi 2-3 business logic yang lebih kompleks.

### Requirements
1. **Pilih Business Logic untuk Diimplementasikan**
   - Data aggregation (misal: total sales per customer)
   - Conditional updates (misal: update stock hanya jika cukup)
   - Complex validation (misal: validate business rules)
   - Data transformation (misal: calculate derived fields)

2. **Implementasi di Service Layer atau Controller**
   - Buat service class jika perlu
   - Implementasi business logic
   - Test logic dengan unit tests atau manual testing
   - Document logic dengan comments

3. **Test dengan Postman**
   - Test business logic dengan valid input
   - Test dengan edge cases
   - Verify output sesuai expected
   - Document test cases

### Deliverables
- [ ] Business logic implementation
- [ ] Test cases dan results
- [ ] Documentation untuk logic
- [ ] Postman collection dengan test requests

## Tips

### Error Handling
- Use specific exception types
- Return user-friendly error messages
- Log technical errors untuk debugging
- Don't expose sensitive information di error messages

### Swagger Documentation
- Be descriptive dengan comments
- Include examples untuk complex types
- Group related endpoints
- Use proper HTTP methods

### Business Logic
- Keep logic separate from controller jika kompleks
- Test edge cases thoroughly
- Document assumptions dan constraints
- Consider performance implications
