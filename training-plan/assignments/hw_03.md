## Tugas 1: Lengkapi Semua Controllers (Wajib)

Semua entity harus memiliki Controller yang mengikuti pola standar.

### Persyaratan per Controller:
1. **Inject Service** melalui constructor (bukan DbContext langsung)
2. **Semua endpoint return** `ApiResponse<T>` — tidak boleh ada yang return Entity langsung
3. **Status code yang tepat**:
   - GET → 200 OK
   - POST → 201 Created (gunakan `CreatedAtAction`)
   - PUT → 200 OK
   - DELETE → 200 OK atau 204 No Content
4. **XML Comment** minimal di setiap method (`/// <summary>`)

### Checklist per Controller:
- [ ] `GET /api/{entity}` → return `ApiResponse<IEnumerable<XxxDto>>`
- [ ] `GET /api/{entity}/{id}` → return `ApiResponse<XxxDto>` (404 jika tidak ada)
- [ ] `POST /api/{entity}` → return `ApiResponse<XxxDto>` dengan status 201
- [ ] `PUT /api/{entity}/{id}` → return `ApiResponse<XxxDto>`
- [ ] `DELETE /api/{entity}/{id}` → return `ApiResponse<object>`

### Verifikasi:
- Buka Swagger UI (`http://localhost:5xxx/swagger`)
- Semua endpoint muncul dengan dokumentasi
- Test setiap endpoint — tidak ada yang error 500
- Test skenario: GET by ID yang tidak ada → harus return 404 dengan pesan jelas

### Deliverable:
- Semua entity memiliki controller lengkap
- Screenshot Swagger UI menampilkan semua endpoint

## Tugas 2: Tambahkan Business Logic (Wajib)

Setiap service harus memiliki minimal **2 business rule** yang meaningful.

### Contoh Business Rules yang Umum di LKS:

**Validasi:**
- Email harus unik (cek duplikat sebelum create)
- Nama tidak boleh kosong setelah di-trim
- Harga tidak boleh negatif atau nol
- Quantity harus bilangan positif

**Conditional Logic:**
- Stok tidak boleh berkurang di bawah nol
- Order tidak bisa dibuat jika customer tidak aktif
- Diskon tidak boleh melebihi 90% dari harga

**Kalkulasi:**
- Total = Price × Quantity
- Harga setelah diskon = Price × (1 - DiscountRate)

### Cara Implementasi:
```csharp
// Di Service layer — BUKAN di Controller
public async Task<OrderDto> CreateOrderAsync(CreateOrderRequest request)
{
    // Business Rule 1: Customer harus ada
    var customer = await _customerRepo.GetByIdAsync(request.CustomerId)
        ?? throw new NotFoundException($"Customer {request.CustomerId} tidak ditemukan");

    // Business Rule 2: Product harus ada dan stok cukup
    var product = await _productRepo.GetByIdAsync(request.ProductId)
        ?? throw new NotFoundException($"Product {request.ProductId} tidak ditemukan");

    if (product.Stock < request.Quantity)
        throw new BusinessException($"Stok tidak cukup. Tersedia: {product.Stock}");

    // Business Rule 3: Kalkulasi total otomatis
    var total = product.Price * request.Quantity;

    // Buat order
    var order = new Order { ... , Total = total };
    await _orderRepo.AddAsync(order);

    // Update stok
    product.Stock -= request.Quantity;
    await _productRepo.UpdateAsync(product);

    return MapToDto(order);
}
```

### Deliverable:
- Minimal 2 business rule per entity yang diimplementasikan
- Test di Swagger bahwa business rule bekerja (contoh: coba buat order dengan stok 0)

## Tugas 3: Lengkapi Custom Exceptions (Wajib)

Buat semua custom exception di `Domain/Exceptions/` dan pastikan semua di-handle di middleware:

```csharp
// NotFoundException.cs
public class NotFoundException : Exception
{
    public NotFoundException(string message) : base(message) { }
}

// ValidationException.cs
public class ValidationException : Exception
{
    public ValidationException(string message) : base(message) { }
}

// ConflictException.cs (untuk duplikat data)
public class ConflictException : Exception
{
    public ConflictException(string message) : base(message) { }
}

// BusinessException.cs (untuk business rule violation)
public class BusinessException : Exception
{
    public BusinessException(string message) : base(message) { }
}
```

Pastikan di `ExceptionHandlingMiddleware.cs`:
- `NotFoundException` → 404
- `ValidationException` → 400
- `ConflictException` → 409
- `BusinessException` → 422

### Deliverable:
- 4 custom exception ada di Domain layer
- Middleware menangani semua exception dengan status code yang benar
- Test di Swagger bahwa setiap exception menghasilkan response dan status code yang benar
