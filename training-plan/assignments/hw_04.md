## Tugas 1: Lengkapi CRUD WinForm via API (Wajib)

Implementasikan CRUD lengkap untuk minimal **2 entity** di WinForm, semua melalui API (bukan langsung ke database).

### Persyaratan:
1. **DataGridView** untuk setiap entity — load data dari `GET /api/{entity}` saat form dibuka
2. **Tombol Tambah** → buka form input → `POST` ke API → refresh DataGridView
3. **Tombol Edit** → pilih baris → isi form dengan data yang ada → `PUT` ke API → refresh
4. **Tombol Hapus** → pilih baris → konfirmasi MessageBox → `DELETE` ke API → refresh
5. **Semua API call menggunakan `async/await`** — tidak ada `.Result` atau `.Wait()`
6. **Error handling**: tampilkan `MessageBox.Show()` jika API tidak bisa diakses atau return error

### Checklist WinForm:
- [ ] DataGridView load otomatis saat form dibuka
- [ ] Tombol Refresh untuk reload data manual
- [ ] Tombol Tambah berfungsi (POST ke API)
- [ ] Tombol Edit berfungsi (PUT ke API) — data pre-filled dari baris yang dipilih
- [ ] Tombol Hapus berfungsi (DELETE ke API) dengan konfirmasi
- [ ] Tidak ada UI freeze saat memanggil API
- [ ] Pesan error user-friendly (bukan stack trace)

### Fitur Tambahan (Opsional):
- TextBox pencarian untuk filter data di DataGridView
- ComboBox untuk dropdown yang load datanya dari API
- Format kolom (currency untuk harga, tanggal yang readable)

### Deliverable:
- WinForm berjalan, CRUD via API berfungsi untuk minimal 2 entity
- Screenshot/video singkat demo CRUD

## Tugas 2: Lengkapi Android — GET + POST (Wajib)

Implementasikan minimal 2 fitur di Android Kotlin yang consume API.

### Persyaratan:
1. **List Screen** untuk minimal 2 entity:
   - `LazyColumn` load data dari `GET /api/{entity}` saat screen dibuka
   - Tampilkan minimal 3 field per item
   - Tombol/gesture Refresh untuk reload data

2. **Add Screen** untuk minimal 1 entity:
   - Form input dengan `TextField` atau `OutlinedTextField` untuk setiap field
   - Tombol Submit → `POST` ke API via ViewModel
   - Setelah berhasil: kembali ke list screen dan refresh data

3. **State Management**:
   - Loading state: tampilkan `CircularProgressIndicator` saat data loading
   - Error state: tampilkan pesan error jika API tidak bisa diakses
   - Empty state: tampilkan pesan jika data kosong

### Checklist Android:
- [ ] Retrofit service terdefinisi untuk semua endpoint yang digunakan
- [ ] ViewModel menggunakan `viewModelScope.launch` untuk coroutine
- [ ] `StateFlow` digunakan untuk state management
- [ ] Loading indicator ditampilkan saat fetch data
- [ ] Error message ditampilkan jika API error
- [ ] POST berhasil → list ter-refresh secara otomatis

### Deliverable:
- Android berjalan di emulator, list data tampil dari API
- Minimal 1 form POST berfungsi

## Catatan untuk Siswa

> **Reminder penting**: Saat Android Emulator mengakses API di localhost, gunakan `10.0.2.2` bukan `localhost` atau `127.0.0.1`.
