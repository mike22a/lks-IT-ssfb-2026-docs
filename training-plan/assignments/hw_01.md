## Tugas 1: Selesaikan Domain Layer (Wajib)

Lengkapi Domain layer dari skeleton project yang sudah dibuat di sesi 1.

### Sub-tugas:
1. **Buat minimal 3 Entity** sesuai domain bisnis yang dipilih bersama mentor
   - Contoh domain "Inventory": `Product`, `Category`, `Transaction`
   - Contoh domain "Sales": `Customer`, `Product`, `Order`, `OrderItem`
   - Setiap entity harus punya: `Id` (int), minimal 3-4 property relevan, `CreatedAt` (DateTime)

2. **Buat Generic Interface** `IRepository<T>` di `Domain/Interfaces/`
   ```csharp
   public interface IRepository<T> where T : class
   {
       Task<IEnumerable<T>> GetAllAsync();
       Task<T?> GetByIdAsync(int id);
       Task AddAsync(T entity);
       Task UpdateAsync(T entity);
       Task DeleteAsync(int id);
   }
   ```

3. **Buat Specific Interface** per entity (extends `IRepository<T>`)
   ```csharp
   // Contoh:
   public interface IProductRepository : IRepository<Product>
   {
       Task<IEnumerable<Product>> GetByCategoryAsync(int categoryId);
   }
   ```

4. **Verifikasi Build**: Jalankan `dotnet build` — pastikan 0 error

### Deliverable:
- Minimal 3 entity class di `Domain/Entities/`
- `IRepository<T>` di `Domain/Interfaces/`
- Minimal 3 specific interface di `Domain/Interfaces/`
- Screenshot `dotnet build` sukses

## Tugas 2: Rancang Database Schema (Wajib)

Buat ERD (Entity Relationship Diagram) untuk semua entity yang akan digunakan.

### Persyaratan ERD:
- Tampilkan semua entity dan field-nya
- Tampilkan relasi antar entity (One-to-Many, Many-to-Many jika ada)
- Tandai Primary Key (PK) dan Foreign Key (FK)
- Tentukan tipe data untuk setiap field

### Tools yang bisa digunakan:
- draw.io (online, gratis): https://draw.io
- Kertas dan pena (foto kemudian)
- dbdiagram.io (online, gratis): https://dbdiagram.io

### Deliverable:
