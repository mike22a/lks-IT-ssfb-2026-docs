# Learning Checklist

## Digital Library Training Kit

---

## Purpose

Menyediakan checklist untuk siswa dalam tracking progress pembelajaran training kit Digital Library.

---

## Scope

Dokumen ini mencakup:

- Checklist harian
- Progress tracker
- Status completion

---

## Learning Objectives

Setelah menggunakan checklist ini, siswa dapat:

- Tracking progress pembelajaran
- Mengetahui apa yang sudah dipelajari
- Mengetahui apa yang perlu dipelajari
- Mengetahui status completion setiap task

---

## Prerequisites

- Siswa harus mengikuti training kit secara berurutan

---

## Business Rules

- Checklist harus diupdate setiap hari
- Task harus dicentang setelah selesai
- Jangan centang task jika belum benar-benar dipahami

---

## Design / Main Content

### Daily Checklist

#### Hari 1: Setup & Database

**Teori:**
- [ ] Memahami visi dan tujuan project
- [ ] Memahami domain bisnis Digital Library
- [ ] Memahami lingkup fungsional dan non-fungsional
- [ ] Memahami overview arsitektur
- [ ] Memahami technology stack

**Database Design:**
- [ ] Memahami struktur tabel
- [ ] Memahami convention penamaan
- [ ] Memahami relasi antar tabel
- [ ] Memahami primary key dan foreign key
- [ ] Memahami indexes
- [ ] Memahami audit fields

**Practice:**
- [ ] Setup solution dan projects
- [ ] Implement database schema
- [ ] Generate seed data
- [ ] Test database queries

**Homework:**
- [ ] Practice SQL queries tambahan
- [ ] Review dokumentasi EF Core
- [ ] Practice database design

#### Hari 2: API Foundation

**Teori:**
- [ ] Memahami Entity Framework Core
- [ ] Memahami Repository Pattern
- [ ] Memahami Dependency Injection
- [ ] Memahami Clean Architecture

**Practice:**
- [ ] Setup EF Core
- [ ] Create domain entities
- [ ] Implement repository pattern
- [ ] Implement Category CRUD API

**Homework:**
- [ ] Implement Book entities
- [ ] Implement Book repository
- [ ] Implement Book API CRUD
- [ ] Test API dengan Postman

#### Hari 3: API Lanjutan

**Teori:**
- [ ] Memahami DTOs
- [ ] Memahami AutoMapper
- [ ] Memahami Service layer
- [ ] Memahami business logic implementation

**Practice:**
- [ ] Create DTOs
- [ ] Implement services
- [ ] Implement User API
- [ ] Implement Borrowing API dengan business rules

**Homework:**
- [ ] Test semua API endpoints
- [ ] Implement error handling
- [ ] Implement logging

#### Hari 4: WinForms Desktop

**Teori:**
- [ ] Memahami WinForms basics
- [ ] Memahami controls common
- [ ] Memahami HttpClient
- [ ] Memahami async/await di WinForms

**Practice:**
- [ ] Setup WinForms project
- [ ] Create main form dengan navigation
- [ ] Implement Category form
- [ ] Implement Book form

**Homework:**
- [ ] Implement Borrowing form
- [ ] Implement search/filter
- [ ] Implement validation

#### Hari 5: Android Mobile

**Teori:**
- [ ] Memahami Android architecture
- [ ] Memahami Retrofit
- [ ] Memahami RecyclerView
- [ ] Memahami MVVM

**Practice:**
- [ ] Setup Android project
- [ ] Implement Retrofit service
- [ ] Implement Book list activity
- [ ] Implement Book detail activity

**Homework:**
- [ ] Implement Borrowing history activity
- [ ] Implement search functionality
- [ ] Final testing dan bug fixing

## Implementation Notes

- Update checklist setiap hari
- Centang task hanya setelah benar-benar selesai
- Tambahkan catatan jika ada masalah atau pertanyaan
- Gunakan checklist untuk diskusi dengan mentor

---

## Common Mistakes

- Tidak update checklist secara teratur
- Centang task padahal belum selesai
- Tidak tambahkan catatan untuk masalah
- Tidak gunakan checklist untuk tracking progress

---

## Exercises

1. Update checklist setiap hari
2. Tambahkan catatan untuk task yang sulit
3. Gunakan checklist untuk diskusi dengan mentor
4. Review checklist sebelum sesi berikutnya

---

## Homework

1. Update checklist harian
2. Review progress sebelum tidur
3. Identifikasi task yang perlu fokus
4. Siapkan pertanyaan untuk mentor
